# 实验 01：用户身份和基础权限

这个实验的目标是把“用户是谁”和“权限怎么影响操作”串成一个完整流程。

完成后你应该能做到：

- 看出自己当前是谁
- 看懂文件和目录的权限位
- 自己修改一个文件和目录的权限
- 理解文件权限和目录权限的不同
- 形成一套基础的权限排查思路

## 实验准备

建议在家目录下创建一个练习区：

```bash
cd ~
mkdir -p linux-perm-lab
cd linux-perm-lab
pwd
```

如果你现在用的是 macOS，也可以先完成绝大部分内容。个别命令输出格式和 Linux 可能略有差异，但核心概念一样。

## 第 1 步：确认当前身份

```bash
whoami
id
groups
```

你应该观察：

- 当前用户名是什么
- 当前用户 ID 和组信息是什么
- 你属于哪些组

## 第 2 步：创建练习文件和目录

```bash
mkdir scripts docs
touch notes.txt run.sh
echo "hello permissions" > notes.txt
echo 'echo "running script"' > run.sh
ls -l
ls -ld scripts docs
```

你应该观察：

- 新文件和新目录的默认权限是什么
- `notes.txt` 和 `run.sh` 大概率一开始都没有执行权限

## 第 3 步：观察文件和目录权限的差别

先看文件：

```bash
ls -l notes.txt run.sh
```

再看目录：

```bash
ls -ld scripts docs
```

思考：

- 为什么目录常见权限会像 `drwxr-xr-x`
- 为什么普通文本文件常见权限不像目录那样带执行位

## 第 4 步：给脚本添加执行权限

```bash
chmod u+x run.sh
ls -l run.sh
./run.sh
```

你应该观察：

- `u+x` 之后，所有者多了执行权限
- 现在脚本可以直接执行了

## 第 5 步：用数字方式设置权限

```bash
chmod 644 notes.txt
chmod 755 scripts
ls -l notes.txt
ls -ld scripts
```

你应该理解：

- `644` 更像普通文件常见权限
- `755` 更像目录常见权限

## 第 6 步：体验目录权限的重要性

先把目录权限改严一点：

```bash
chmod 700 docs
ls -ld docs
```

再在里面创建文件：

```bash
touch docs/private.txt
ls docs
```

这里的重点不是制造复杂报错，而是让你观察目录权限位本身。

如果你在多用户 Linux 环境里，这一步会更容易体现“别人能不能进入这个目录”。

## 第 7 步：观察更详细属性

```bash
stat notes.txt
stat scripts
```

你应该观察：

- 权限信息
- 所有者和所属组
- 时间属性

## 第 8 步：看默认权限掩码

```bash
umask
umask -S
```

你不需要马上把它算得很熟，但要建立印象：

- 新文件默认权限不是随便来的
- `umask` 会影响默认创建结果

## 第 9 步：建立一套排查顺序

现在请你自己对着当前目录做一次小复盘：

```bash
whoami
ls -l
ls -ld docs scripts
```

然后回答：

1. 我现在是谁
2. 哪些对象是我创建的
3. 哪些对象有执行权限
4. 哪些对象是目录
5. 如果某一步提示权限不够，我会先看哪里

## 可选步骤：观察 `sudo` 的使用场景

如果你在真正的 Linux 环境里，并且有管理员权限，可以额外观察：

```bash
sudo ls /root
```

这一步的重点不是多执行几次 `sudo`，而是理解：

- 有些目录不是普通用户该直接访问的
- `sudo` 是临时提权，不是默认工作方式

## 实验完成标准

如果你已经能做到下面这些，说明这个实验达标了：

- 能看懂 `ls -l` 里的所有者、所属组和权限位
- 能给脚本增加执行权限
- 能区分文件权限和目录权限
- 能理解 `644` 和 `755` 的常见含义
- 遇到权限问题时，知道先查身份、归属和权限位

## 实验后复盘

建议你自己再回答这 5 个问题：

1. 为什么 `run.sh` 一开始通常不能直接执行？
2. 为什么目录的 `x` 权限和文件的 `x` 权限意义不同？
3. 如果一个文件属于 `root`，普通用户最常见会卡在哪？
4. 为什么 `sudo` 不应该成为默认习惯？
5. 以后看到 `Permission denied`，你的第一反应应该是什么？
