# Linux 学习总览

## 学习目标

这一套路线的目标不是只会背命令，而是逐步做到：

- 看得懂 Linux 系统的基本结构
- 能在终端中独立完成常见操作
- 能排查基础系统问题
- 能编写简单 Bash 脚本提高效率
- 能搭建个人 Linux 知识体系

## 适合人群

- Linux 零基础或基础不系统
- 想学服务器运维、后端开发、云原生前置知识
- 想提升终端操作和系统理解能力

## 推荐学习方法

每个知识点按下面顺序学：

1. 先理解概念
2. 再自己跑命令
3. 然后记录结果和踩坑
4. 最后做一个小练习巩固

建议每学完一个主题，都补三样内容：

- `notes.md`: 概念和原理
- `commands.md`: 常用命令和示例
- `questions.md`: 易错点、疑问、复盘

## 推荐环境

如果你是本地学习，建议至少准备一种 Linux 使用环境：

- 虚拟机中的 Ubuntu
- 云服务器中的 Linux 主机
- macOS 上通过终端模拟基础命令练习

更推荐你有一台真正的 Linux 环境，这样更适合学习：

- 用户和权限
- systemd 和服务管理
- 网络排查
- 包管理
- Shell 脚本自动化

## 学习原则

- 不追求一次学很多，重在连续性
- 看到命令一定要亲手敲一遍
- 学完一块就做实验，不要只看不练
- 命令记不住很正常，重点是知道去哪里查
- 遇到错误信息先读懂，再搜索，再记录

## 最终验收标准

学完这套内容后，理想状态下你应该能做到：

- 独立完成文件、文本、权限、进程、网络的基础操作
- 看懂常见系统目录和日志
- 使用 `ssh`、`scp`、`curl`、`ps`、`top`、`grep`、`find`
- 编写一个带参数、带条件判断的 Bash 脚本
- 对一个简单 Linux 问题进行基础排查并写出排查过程
