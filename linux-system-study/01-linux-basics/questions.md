## 参考答案

### 1. Linux、发行版、Shell、Terminal 分别是什么？

- Linux 常常指 Linux 内核
- 发行版是基于 Linux 内核打包出来的完整系统
- Shell 是解释命令的程序
- Terminal 是显示命令行界面的程序

### 2. 为什么 Linux 学习离不开命令行？

因为服务器环境、排查问题、自动化操作都高度依赖命令行，而且命令行更容易重复执行和组合。

### 3. 绝对路径和相对路径有什么区别？

- 绝对路径从根目录 `/` 开始写
- 相对路径相对于当前工作目录来写

### 4. `.`、`..`、`~` 分别表示什么？

- `.`：当前目录
- `..`：上一级目录
- `~`：当前用户家目录

### 5. `pwd` 的作用是什么？

显示当前工作目录。

### 6. `ls` 和 `ls -la` 的区别是什么？

- `ls` 只列出普通可见内容
- `ls -la` 会显示详细信息并包含隐藏文件

### 7. 为什么输入 `ls` 时系统能找到它？

因为 Shell 会按 `PATH` 里配置的目录顺序去查找可执行程序。

### 8. `PATH` 是做什么的？

告诉 Shell 去哪些目录里找命令。

### 9. `/etc`、`/var`、`/tmp` 大概分别放什么？

- `/etc`：配置文件
- `/var`：变化频繁的数据，比如日志
- `/tmp`：临时文件

### 10. `whoami` 和 `id` 各自更适合回答什么问题？

- `whoami`：我当前是谁
- `id`：我的用户 ID、组 ID 和所属组是什么

### 11. `which` 和 `type` 有什么不同？

- `which` 常用于快速找命令路径
- `type` 更适合判断一个名字到底是外部程序、Shell 内建命令，还是别名

### 12. 当你完全不会一个命令时，最先该怎么查？

先看 `命令 --help`，需要更完整说明时再看 `man 命令`。

### 13. 为什么 Linux 很多操作都离不开路径？

因为很多命令都在围绕文件、目录和位置进行操作，而这些都要靠路径表达。

### 14. `bin` 一般表示什么？

通常指放可执行程序的目录，来自 `binaries` 这个说法。

### 15. 为什么说“Terminal 不是 Shell”？

因为 Terminal 是窗口或界面，Shell 才是真正解释和执行命令的程序。
