# Linux 基础概念笔记

这份笔记的目标是帮你先建立一张“Linux 地图”。很多初学者不是学不会命令，而是一上来没有整体感，所以命令像散点，记不住也连不起来。

## 1. Linux 到底是什么

平时大家说“Linux”，通常会混着说下面几层东西：

- `Linux`：严格说通常指 Linux 内核
- `发行版`：在 Linux 内核外面，再搭配命令工具、包管理器、默认配置后的完整系统
- `Shell`：你输入命令时真正和你交互的命令解释器
- `Terminal`：显示命令行界面的程序窗口

可以这样理解：

- 内核负责管理硬件、内存、进程、磁盘、网络这些底层资源
- Shell 负责理解你输入的命令
- Terminal 负责把这次交互显示出来

例如你输入：

```bash
pwd
```

背后大致发生的是：

1. Terminal 把你输入的内容显示出来
2. Shell 识别出你要执行 `pwd`
3. 系统找到 `pwd` 对应程序
4. 程序获取当前工作目录
5. 结果再回到终端界面

## 2. 为什么学 Linux 要先学命令行

因为 Linux 学习不只是“会点图形界面”，而是要逐步理解系统怎么工作。

命令行的重要性在于：

- 很多服务器环境没有完整图形界面
- 命令行更容易重复执行和自动化
- 日志、网络、进程、权限排查都离不开命令行

所以 Linux 学习的主线其实是：

- 认识系统
- 学会发出命令
- 看懂命令输出
- 把多个命令串成完整操作

## 3. 常见发行版先建立印象就够

发行版可以理解成“不同团队打包出来的 Linux 成品系统”。

常见发行版：

- `Ubuntu`：入门友好，资料多，社区大
- `Debian`：稳定，很多系统以它为基础
- `CentOS`：过去在服务器上很常见
- `Rocky Linux`、`AlmaLinux`：接近传统企业服务器生态
- `Arch Linux`：更强调自己组装和理解系统

初学阶段不需要纠结“学哪个发行版最正宗”，更重要的是先在一个发行版上学会基本概念和命令。

## 4. 你打开终端后，面对的是什么

你在终端里经常会看到一个提示符，例如：

```text
zcy@host:~$
```

这里通常在表达这些信息：

- `zcy`：当前用户
- `host`：主机名
- `~`：当前所在目录是家目录
- `$`：普通用户提示符

如果结尾是 `#`，常常表示当前是 root 用户或高权限环境。

## 5. 命令的基本结构

Linux 命令通常长这样：

```bash
command [options] [arguments]
```

例如：

```bash
ls -l /etc
```

拆开理解：

- `ls`：命令本身
- `-l`：选项，控制显示方式
- `/etc`：参数，也就是这条命令要操作的对象

初学时要先养成这个思路：

- 命令名说明“做什么”
- 选项说明“怎么做”
- 参数说明“对谁做”

## 6. 路径是 Linux 学习最重要的基础之一

### 6.1 当前目录

你在终端里总是“站在某个目录里”，这个位置叫当前工作目录。

查看方法：

```bash
pwd
```

### 6.2 绝对路径和相对路径

绝对路径：从根目录 `/` 开始写的完整路径。

例如：

```bash
/home/zcy/projects
```

相对路径：相对于当前目录写的路径。

例如：

```bash
./notes
../downloads
```

常见符号：

- `/`：根目录
- `.`：当前目录
- `..`：上一级目录
- `~`：当前用户家目录

### 6.3 为什么路径这么重要

因为几乎所有命令都在处理路径：

- 你在哪里
- 你要去哪里
- 你要操作哪个文件或目录

如果路径没搞懂，后面很多文件、权限、脚本问题都会卡住。

## 7. Linux 常见目录先知道用途

初学阶段，不要求你背完整文件系统规范，但下面这些目录最好先认识：

- `/`：整个文件系统起点
- `/home`：普通用户家目录
- `/root`：root 用户家目录
- `/etc`：系统和服务配置文件
- `/var`：经常变化的数据，比如日志
- `/tmp`：临时文件
- `/usr`：很多程序和共享资源
- `/bin`：基础命令
- `/sbin`：偏系统管理的命令
- `/opt`：额外安装的软件

一个很实用的理解方式是：

- 看配置，多半会想到 `/etc`
- 看日志，多半会想到 `/var/log`
- 看用户自己的文件，多半会想到 `/home/用户名`

注意：不同系统会有细节差异，尤其是 macOS 和 Linux 的目录布局并不完全一样，但这些概念在 Linux 学习里非常核心。

## 8. “一切皆文件”要怎么理解

这句话不是说所有东西看起来都像文本文件，而是说 Linux 倾向于用“文件接口”的方式统一处理很多对象。

例如：

- 普通文档是文件
- 目录是一种特殊文件
- 设备也常通过文件形式暴露接口
- 配置和日志通常都以文本文件存在

所以 Linux 学习里很多事情最后都会回到：

- 找文件
- 看文件
- 改文件
- 改文件权限

## 9. Shell 是什么

Shell 是命令解释器，你在终端里输入的内容，通常先交给 Shell 处理。

常见 Shell：

- `bash`
- `zsh`
- `sh`

你可以用下面命令看看当前用的是哪个 Shell：

```bash
echo $SHELL
```

这里先建立一个印象就够：Terminal 是窗口，Shell 才是理解命令的人。

## 10. PATH 是什么

当你输入：

```bash
ls
```

系统并不是凭空知道 `ls` 在哪里，而是会去 `PATH` 指定的一组目录里找这个命令。

查看方法：

```bash
echo $PATH
```

它通常是一串用 `:` 分隔的目录。

这意味着：

- 如果命令所在目录在 `PATH` 里，你就可以直接输入命令名
- 如果不在，你要么写完整路径，要么把它加入 `PATH`

这就是为什么很多命令都在各种 `bin` 目录里。

## 11. 如何自学陌生命令

Linux 学习不是把所有命令背下来，而是知道怎么查。

最常用的两个入口：

```bash
ls --help
man ls
```

你可以先记住：

- `--help`：看简要帮助
- `man`：看更完整的手册

这是 Linux 学习中非常值得尽早形成的习惯。

## 12. 初学者最容易混淆的几个点

### Shell 和 Terminal 不是一回事

- Terminal 是窗口
- Shell 是解释命令的程序

### Linux 和发行版不是一回事

- Linux 常指内核
- Ubuntu、Debian 这些是发行版

### 相对路径不是“短路径”

相对路径不是长度短，而是“相对于当前目录来写”。

### `bin` 不是某种单独文件格式

`bin` 更常指目录名，意思是放可执行程序的地方。

## 13. 这一章学完后你应该留下什么印象

如果你学完这章，脑子里至少应该有这几个稳定概念：

- Linux 是一个以命令行和文件系统为核心的系统
- 很多操作都围绕路径、文件和进程展开
- 命令通常由命令名、选项、参数组成
- 目录结构和帮助系统是后续学习的基础

如果这些印象已经建立起来，后面学命令时就不会那么散。
