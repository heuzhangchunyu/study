# 实验 01：第一次接触 Linux 环境

这个实验的目标不是做复杂操作，而是让你第一次真正“看见系统”。

完成后你应该能回答：

- 我当前是谁
- 我现在在哪
- 我在用什么 Shell
- 我的系统大概长什么样
- 我遇到陌生命令时该怎么查

## 实验准备

如果你有 Linux 主机，优先在 Linux 上做。

如果你暂时只有 macOS，也可以先完成大部分内容。个别命令输出会和 Linux 不完全一样，但学习方法一致。

## 第 1 步：确认当前用户和位置

运行：

```bash
whoami
pwd
```

记录：

- 当前用户名是什么
- 当前工作目录是什么

思考：

- 如果我换一个用户登录，这里会不会变
- 为什么后面的相对路径都依赖 `pwd` 的结果

## 第 2 步：看看当前目录里有什么

运行：

```bash
ls
ls -la
```

观察：

- `ls` 和 `ls -la` 输出有什么差别
- 有没有以 `.` 开头的隐藏文件

思考：

- 为什么很多配置文件默认是隐藏的

## 第 3 步：切换几个典型目录

运行：

```bash
cd /
pwd
ls
cd ~
pwd
cd ..
pwd
cd -
pwd
```

你应该搞清楚：

- `/` 是根目录
- `~` 是家目录
- `..` 是上一级目录
- `-` 可以回到上一次所在目录

## 第 4 步：确认当前 Shell 和 PATH

运行：

```bash
echo $SHELL
echo $PATH
```

观察：

- 当前 Shell 是什么
- `PATH` 里有哪些目录
- 这些目录之间是用什么符号分隔的

思考：

- 为什么输入 `ls` 时不用写完整路径

## 第 5 步：找到几个命令来自哪里

运行：

```bash
which ls
which bash
type cd
type -a ls
```

观察：

- `ls` 在哪里
- `cd` 是不是外部程序
- 某个命令是否可能有多个来源

## 第 6 步：看看系统基本信息

运行：

```bash
hostname
uname
uname -a
date
```

如果你在 Linux 上，再试：

```bash
cat /etc/os-release
```

你不需要把所有输出背下来，但要知道这些命令分别在回答什么问题。

## 第 7 步：认识常见系统目录

运行：

```bash
ls /
```

然后重点观察这些名字：

- `home`
- `root`
- `etc`
- `var`
- `tmp`
- `usr`
- `bin`

练习：

试着用一句话写下你现在对它们的理解。哪怕不完全准确也没关系，关键是先建立印象。

## 第 8 步：学会自己查帮助

运行：

```bash
pwd --help
ls --help
man pwd
```

如果 `man` 打开了分页界面，试试这些操作：

- `Space` 向下翻页
- `/directory` 搜索关键字
- `q` 退出

## 第 9 步：写一个自己的入门总结

请你自己写下这 5 句话：

1. Linux 不只是一个命令集合，它更像什么？
2. Shell 和 Terminal 有什么区别？
3. 绝对路径和相对路径有什么区别？
4. `PATH` 的作用是什么？
5. 遇到陌生命令时，我下一步该怎么做？

## 实验完成标准

如果你已经能独立完成下面这些，说明这个实验达标了：

- 能切换目录并说清楚当前目录位置
- 能解释 `PATH` 的基本作用
- 能区分 Shell 和 Terminal
- 能说出几个常见系统目录的大致用途
- 能自己用 `--help` 或 `man` 查一个陌生命令
