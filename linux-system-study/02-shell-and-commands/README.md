# 02 Shell And Commands

这一章的重点是把“会看命令”推进到“会用命令”。

学完这一章后，你应该能做到：

- 理解 Shell 在执行命令时到底做了什么
- 看懂命令的结构：命令名、选项、参数
- 熟练完成文件和目录的基础操作
- 理解通配符、引号、重定向、管道这些常见写法
- 能把几条简单命令串成一段完整操作流程

## 建议学习顺序

1. 先看 `notes.md`
2. 再练 `commands.md`
3. 然后做 `labs/01-command-workflow.md`
4. 最后用 `questions.md` 自测

## 文件说明

- `notes.md`: Shell 工作方式和命令结构
- `commands.md`: 这一章要重点练习的命令和写法
- `questions.md`: 复盘题和参考答案
- `labs/01-command-workflow.md`: 第一轮命令组合实验

## 本章建议重点关注

- 命令后面的内容到底是选项还是参数
- `*` 为什么能一次匹配多个文件
- `"` 和 `'` 为什么结果不一样
- `>`、`>>`、`|` 各自到底在干什么
- `cp` 和 `mv` 的区别是什么

## 本章完成标准

如果下面这些你都能做到，说明这一章学得比较稳：

- 能解释 `command [options] [arguments]` 这个结构
- 能独立使用 `mkdir`、`touch`、`cp`、`mv`、`rm`
- 能看懂并使用 `*`、`?` 这样的通配符
- 能说清 `>`、`>>`、`|` 的区别
- 能用几条命令完成一次简单文件整理流程
