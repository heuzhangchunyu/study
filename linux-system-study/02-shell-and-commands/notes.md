# Shell 和命令基础笔记

这一章的目标是把命令行从“像咒语”变成“有结构的工具”。

如果说上一章是在认识 Linux 这个系统，这一章就是开始真正学会和它打交道。

## 1. Shell 到底在做什么

你在终端里输入一行命令后，不是直接由 Terminal 执行，而是先交给 Shell 处理。

Shell 大致会做这些事情：

1. 读取你输入的一整行命令
2. 按规则拆成命令名、选项、参数
3. 处理变量、通配符、引号这些特殊写法
4. 找到要执行的命令
5. 运行命令，并把输出结果显示出来

例如你输入：

```bash
ls -l *.md
```

Shell 大致会这样理解：

- `ls` 是要执行的命令
- `-l` 是选项
- `*.md` 会先被 Shell 展开成当前目录下所有 `.md` 文件

这也是为什么学习命令行时，不能只记命令名，还要理解 Shell 的处理规则。

## 2. 命令的基本结构

Linux 命令通常长这样：

```bash
command [options] [arguments]
```

例如：

```bash
cp notes/todo.txt notes/todo.bak
```

可以拆成：

- `cp`：命令名，表示复制
- `notes/todo.txt`：源文件路径
- `notes/todo.bak`：目标文件路径

再看一个有选项的例子：

```bash
mkdir -p project/logs/archive
```

这里：

- `mkdir`：创建目录
- `-p`：选项，表示父目录不存在也一起创建
- `project/logs/archive`：目标路径

一个实用记忆法：

- 命令名回答“做什么”
- 选项回答“怎么做”
- 参数回答“对谁做”

## 3. 选项的常见写法

命令选项最常见的两种形式：

- 短选项：`-l`、`-a`、`-r`
- 长选项：`--help`、`--version`

有些短选项可以合并写：

```bash
ls -l -a
ls -la
```

这两种写法通常等价。

但不是所有命令都支持完全相同的选项，也不是所有系统都支持一样的长选项，所以遇到不确定的情况，优先查 `--help`。

## 4. 参数和路径是命令的主要对象

很多命令操作的其实都是路径。

例如：

```bash
touch notes/todo.txt
mv notes/todo.txt backup/todo.txt
rm backup/todo.txt
```

这里的参数都是路径，只是命令对它们做的事情不同：

- `touch`：创建文件或更新时间戳
- `mv`：移动或重命名
- `rm`：删除

所以命令行学习里，路径感非常重要。你必须经常知道：

- 我现在在哪
- 我要操作谁
- 操作后它会去哪

## 5. Shell 内建命令和外部命令

不是所有命令都是磁盘上的独立程序。

例如：

- `cd` 常常是 Shell 内建命令
- `ls` 常常是外部程序

为什么 `cd` 经常做成内建命令？

因为切换当前目录这件事，必须直接改变当前 Shell 自己的状态。如果它只是启动一个外部程序，那个程序结束后，当前 Shell 的目录并不会真的改变。

可以用下面命令查看：

```bash
type cd
type ls
```

初学阶段不用纠结所有细节，但要先知道这两类命令都存在。

## 6. 引号为什么重要

Shell 会把空格当成分隔符，所以带空格的内容如果不加处理，很容易被拆错。

例如：

```bash
mkdir my notes
```

这不是创建一个叫 `my notes` 的目录，而是尝试创建两个目录：

- `my`
- `notes`

如果你真的想创建带空格的名字，需要这样写：

```bash
mkdir "my notes"
mkdir 'my notes'
mkdir my\ notes
```

### 双引号 `"`

双引号会保留整体，同时仍然允许变量展开。

例如：

```bash
echo "home is $HOME"
```

Shell 会先把 `$HOME` 替换成实际值。

### 单引号 `'`

单引号会把内容尽量原样保留，不做变量展开。

例如：

```bash
echo 'home is $HOME'
```

这里输出的就是字面量 `$HOME`。

### 什么时候最容易出问题

这些场景很常见：

- 路径里有空格
- 文本里有 `$`、`*` 这些特殊字符
- 你希望某段内容被原样写入文件

## 7. 通配符是什么

通配符也叫 glob，用来一次匹配多个文件。

最常见的几个：

- `*`：匹配任意长度字符
- `?`：匹配单个字符
- `[abc]`：匹配括号里任意一个字符

例如：

```bash
ls *.md
rm *.log
cp file?.txt backup/
```

含义分别是：

- `*.md`：所有以 `.md` 结尾的文件
- `*.log`：所有以 `.log` 结尾的文件
- `file?.txt`：像 `file1.txt`、`fileA.txt` 这种单字符占位的文件

要注意一点：

- 通配符通常是先由 Shell 展开，再把结果交给命令
- 如果你没看清当前目录里有哪些文件，`rm *.log` 这类命令就可能删掉超出预期的东西

## 8. 重定向：把输入输出接到别的地方

很多命令都会把结果输出到屏幕，这个默认输出位置叫标准输出，也就是 stdout。

你也可以把输出改写到文件里，这就叫重定向。

### 覆盖写入 `>`

```bash
echo "hello" > hello.txt
```

含义：

- 如果文件不存在，就创建
- 如果文件已存在，就覆盖原内容

### 追加写入 `>>`

```bash
echo "second line" >> hello.txt
```

含义：

- 不覆盖原内容
- 新内容接在文件末尾

### 标准错误 `2>`

除了标准输出，还有标准错误，也就是 stderr。

例如：

```bash
ls not_exist.txt 2> error.txt
```

这里报错信息不会显示到屏幕，而是写进 `error.txt`。

初学阶段先记住一个结论：

- `>` 和 `>>` 常用于正常输出
- `2>` 常用于错误输出

## 9. 管道：把一个命令的输出交给下一个命令

管道符是：

```bash
|
```

例如：

```bash
ls -l | grep notes
```

它的意思不是“先执行再显示”，而是：

- 先让 `ls -l` 产生输出
- 再把这份输出交给 `grep notes` 继续处理

你可以把管道理解成一条流水线。

另一个例子：

```bash
history | tail -n 10
```

意思是：

- 先拿到命令历史
- 再只看最后 10 条

一个很关键的直觉是：

- 重定向是在改输出去哪里
- 管道是在把输出交给别的命令继续处理

## 10. 命令组合：不只是一条一条敲

除了管道，Shell 里还经常会用这些组合方式。

### 连续执行 `;`

```bash
pwd; ls
```

无论前一条成功还是失败，后一条都会执行。

### 成功后再执行 `&&`

```bash
mkdir notes && cd notes
```

含义：

- 先创建目录
- 只有创建成功，才进入这个目录

### 失败后再执行 `||`

```bash
cd notes || echo "directory not found"
```

含义：

- 如果 `cd notes` 失败
- 才执行后面的提示

初学阶段最常用的是 `&&`，因为它很适合把一段依赖前一步成功的操作串起来。

## 11. 命令历史和补全很值得尽早养成习惯

### 历史记录

你以前敲过的命令可以用：

```bash
history
```

查看。

配合方向键向上，也能快速找回刚才敲过的命令。

### Tab 补全

如果路径或命令太长，不要强行手打。

试着输入一部分后按 `Tab`：

- Shell 会尝试自动补全
- 或者显示可选项

这能显著减少拼写错误。

## 12. 初学者常见坑

### 误删文件

最典型的是：

```bash
rm *.log
```

在没确认匹配范围前就执行删除。

更稳妥的习惯是先用：

```bash
ls *.log
```

确认会匹配哪些文件。

### 覆盖文件

例如：

```bash
echo "new content" > notes.txt
```

如果你本来只是想追加，就会把原内容清空。

### 忘记引号

只要路径里有空格，就要格外小心。

### 把 `cp` 和 `mv` 混了

- `cp` 复制后原文件还在
- `mv` 通常是移动或改名，原路径会消失

## 13. 这一章学完后应该形成什么感觉

如果这一章学得顺，你会开始有这种感觉：

- 命令不是死记硬背，而是有结构可拆
- Shell 不是“黑盒子”，它有明确的处理规则
- 重定向、管道、通配符这些写法开始有直觉
- 你已经能把几条命令连成一个小流程

到这里，你就真正开始进入命令行使用阶段了。
